✅ จุดเด่นของ Workflow นี้

  • ทำงานพร้อมกัน: 3 คนทำงานพร้อมกันได้โดยไม่รบกวนกัน
  • แยกงานชัดเจน: แต่ละคนรับผิดชอบไฟล์ต่างกัน
  • ป้องกัน Conflict: ใช้ Branch แยกกัน + Sync ก่อน Push
  • มี Code Review: ต้องผ่าน Pull Request ก่อน Merge
  • ปลอดภัย: Branch Protection + CODEOWNERS ป้องกันแก้ไฟล์ผิด
  • ง่ายต่อการ Rollback: ถ้าพังสามารถย้อนกลับได้

⚠️ สิ่งที่ต้องระวัง

  • Pull ก่อนเสมอ: ก่อนเริ่มงานใหม่ ต้อง Pull Staging ก่อน
  • Sync ก่อน Push: ใช้ safe-push.sh เพื่อดึงงานคนอื่นมาผสมก่อน
  • ห้ามแก้ Shared Code: ต้องแจ้ง Team Lead ก่อน
  • ห้าม Push ตรง Staging: ต้องผ่าน Branch + PR เสมอ
  • Test ก่อน Push: ทดสอบให้แน่ใจว่าโค้ดไม่พัง
