🏗️ Backend API Architecture - Before & After

❌ ก่อนแก้ไข (มีปัญหา)

1. Frontend Request

Frontend ส่ง request ไปที่ /api/orders เพื่อดึงข้อมูล Orders ทั้งหมด

2. Backend Response

Backend ส่งข้อมูล Orders ทั้งหมดกลับมา (อาจเป็นพัน records)

3. Frontend Processing

Frontend ต้องคำนวณ Report เอง:
• Group by ประเทศ/Supplier/วันที่
• นับจำนวน Orders
• คำนวณยอดรวม
• หาค่าเฉลี่ย

⚠️ ปัญหาที่เกิด

429 Rate Limit Error
• ดึงข้อมูลมากเกินไป
• ช้า (ต้องรอดาวน์โหลดข้อมูลทั้งหมด)
• ใช้ bandwidth มาก
• ประมวลผลฝั่ง Client ช้า

✅ หลังแก้ไข (แก้ปัญหาแล้ว)

1. Frontend Request

Frontend ส่ง request ไปที่ Report Endpoint เฉพาะที่ต้องการ เช่น /api/reports/by-country

2. Backend Processing

Backend คำนวณ Report ฝั่ง Server:
• ใช้ SQL GROUP BY
• คำนวณ SUM, AVG, COUNT
• จำกัด 100 records
• เรียงลำดับตามที่ต้องการ

3. Backend Response

Backend ส่งผลลัพธ์ที่พร้อมใช้งานกลับมา (เฉพาะข้อมูลที่จำเป็น)

✨ ผลลัพธ์

ไม่มี Error แล้ว!
• เร็วขึ้นมาก
• ใช้ bandwidth น้อย
• ไม่มี Rate Limit
• Frontend แค่แสดงผล

📡 7 Report Endpoints ที่ต้องสร้าง

GET

1. Order Summary

/api/reports/summary
สรุปภาพรวม Orders ทั้งหมด (จำนวน, ยอดรวม, ค่าเฉลี่ย)
GET

2. By Country

/api/reports/by-country
รายงานแยกตามประเทศ (จาก product_snapshot) - จำกัด 100
GET

3. By Supplier

/api/reports/by-supplier
รายงานแยกตาม Supplier - จำกัด 100
GET

4. By Travel Date

/api/reports/by-travel-date
รายงานแยกตามเดือนเดินทาง (ภาษาไทย + พ.ศ.) - จำกัด 100
GET

5. By Booking Date

/api/reports/by-booking-date
รายงานแยกตามเดือนจอง (ภาษาไทย + พ.ศ.) - จำกัด 100
GET

6. Repeat Customers

/api/reports/repeat-customers
ลูกค้าที่จองซ้ำ (มากกว่า 1 ครั้ง) - จำกัด 100
GET

7. Countries List

/api/reports/countries
รายการประเทศทั้งหมด (สำหรับ dropdown) - ไม่จำกัด

🎯 ประโยชน์ที่ได้รับ

⚡

เร็วขึ้นมาก

ไม่ต้องดึงข้อมูลทั้งหมด แค่ดึงผลลัพธ์ที่คำนวณแล้ว

✅

แก้ 429 Error

ไม่มี Rate Limit เพราะไม่ต้องดึงข้อมูลมากเกินไป

📉

ใช้ Bandwidth น้อย

ส่งเฉพาะข้อมูลที่จำเป็น ไม่ส่งข้อมูลดิบทั้งหมด

🎨

Frontend เบาขึ้น

ไม่ต้องคำนวณเอง แค่แสดงผลข้อมูลที่ได้มา

🔒

ปลอดภัยกว่า

ไม่ส่งข้อมูลดิบทั้งหมดไปฝั่ง Client

📊

Scalable

รองรับข้อมูลเยอะขึ้นได้ เพราะคำนวณฝั่ง Server
